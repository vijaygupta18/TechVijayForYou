# Multi-Cloud Architecture: High Level Design

---

## Simple Explanation

See, earlier we used to have everything in one place. Now we have two data centers - let's call them Cloud A and Cloud B.

The user makes a request, but we need to decide - should this go to Cloud A or Cloud B?

That's where the **Routing Layer** comes in picture. It sits on top of both clouds and acts like a traffic police - directing each request to the right place.
